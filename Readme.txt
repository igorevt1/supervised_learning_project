Исходная задача состояла в следующем: необходимо спрогнозировать уход клиента из банка в ближайшее время. Для этого необходима модель, способная достаточно хорошо классифицировать таких клиентов. Качество модели предлагается измерять значением `F1 меры` - ее значение для итоговой модели должно быть не ниже 0.59.

Исходные данные: исторические данные банка - данные о клиентах и отметки о том, остались они клиентами банка или нет. В данных есть дисбаланс классов: клиентов, раасторгнувших договор, меньше, чем оставшихся в банке.

Источник данных: [https://www.kaggle.com/barelydedicated/bank-customer-churn-modeling](https://www.kaggle.com/barelydedicated/bank-customer-churn-modeling)

Используемые в проекте модели и среднее значение `F1 меры` при разных способах борьбы с дисбалансом для каждой модели:
    - RandomForestClassifier (случайный лес) - среднее значение `F1 меры` = 0.5705561952605972
    - DecisionTreeClassifier (дерево решений) - среднее значение `F1 меры` = 0.5414575895265361
    - LogisticRegression (логистическая регрессия) - среднее значение `F1 меры` = 0.43614630679022665

Итоговая модель: в качестве итоговой выбрана модель `RandomForestClassifier` (случайный лес) с гиперпараметрами `n_estimators` = 60 и `max_depth` = 9. В качестве обучающей выборки для тестовой модели используется обучающая + валидационная выборка с балансировкой классов методом `upsmapling`.

Результат: Метрика `F1 мера` на тестовой выборке достигла значения 0.6331967213114753.
